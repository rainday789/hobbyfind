## 1. 제품 상세 설명 (Detailed Product Description)

### 1.1 서비스 개요

**HobbyFind**는 운동형·지능형·예술형 3가지 카테고리로 분류된 **18개 고정 취미**를 탐색하고, 회원이 관심 취미를 **북마크**하여 관리할 수 있는 웹 기반 취미 추천 사이트이다.

| 이용자 | 이용 가능 범위 |
|--------|----------------|
| **비회원** | 홈(루트) 및 카테고리별 페이지 **탐색** |
| **회원** | 탐색 + **북마크** + **마이페이지**(북마크 목록·통계) |

### 1.2 핵심 목표

| 목표 | 설명 |
|------|------|
| **탐색** | 카테고리·그리드 UI로 18개 취미를 직관적으로 찾도록 지원 |
| **저장** | 로그인 사용자가 관심 취미를 북마크로 저장 |
| **분석** | 마이페이지에서 카테고리별 북마크 분포를 시각화 |
| **접근성** | 비회원도 콘텐츠 탐색 가능, 북마크 시점에만 로그인 유도 |

### 1.3 취미 데이터 (고정, 변경·확장 불가)

| 카테고리 | 취미 (6종) |
|----------|------------|
| **운동형** | 조깅/러닝, 요가, 수영, 자전거, 클라이밍, 댄스 |
| **지능형** | 독서, 퍼즐, 체스, 프로그래밍, 외국어 학습, 사진 촬영 |
| **예술형** | 그림 그리기, 악기 연주, 요리, 서예, 도자기 만들기, 정원 가꾸기 |

- 총 **18개** 고정
- 강의 실습용 제한으로 **추가·삭제·수정 불가**
- 취미별 **상세 페이지 없음** (카드 단위 정보만 제공)

## 2. 참고 서비스 및 근거 (Reference Services with Detailed Rationale)

| 참고 서비스 | 참고 요소 | HobbyFind 적용 근거 |
|-------------|-----------|---------------------|
| **Pinterest** | 이미지 카드, 보드(저장) UX | 취미 카드 그리드 + 북마크(저장) 패턴 차용 |
| **Netflix** | 장르(카테고리) 필터, 그리드 탐색 | 운동형/지능형/예술형 필터 및 목록 UX 참고 |
| **Goodreads** | 읽고 싶은 책 목록, 간단 통계 | 북마크 목록 + 카테고리 분포 통계 구조 참고 |
| **HobbyDB** | 취미 분류·메타데이터 | 3대분류 + 취미명·설명·이미지 데이터 모델 참고 |
| **Notion (카드 뷰)** | 카드형 정보 배치 | 그리드 카드 UI(제목·이미지·설명) 레이아웃 참고 |

> 참고 서비스는 **UX·정보 구조** 벤치마킹용이며, 동일 기능·규모를 목표로 하지 않는다.

## 3. 핵심 기능 및 사양 (Core Features and Specifications)

### 3.1 페이지 구성

#### (1) 루트 페이지 `/`

| 영역 | 사양 |
|------|------|
| **Top Bar** | 로고, 카테고리 링크(운동형·지능형·예술형), 로그인/회원가입(비로그인) 또는 마이페이지/로그아웃(로그인) |
| **Hero 섹션** | 사이트 소개 문구(서비스 목적·이용 방법 요약) |
| **메인 콘텐츠** | 취미 **카드 그리드** (18개 또는 필터 결과) |
| **카테고리 필터** | 전체 / 운동형 / 지능형 / 예술형 — 선택 시 해당 취미만 그리드에 표시 |
| **카테고리 이동** | Top Bar 또는 필터에서 카테고리 선택 시 → **카테고리별 전용 페이지**로 이동 |

**취미 카드 사양**

| 요소 | 필수 여부 | 설명 |
|------|-----------|------|
| 썸네일 이미지 | O | 취미 대표 이미지 |
| 취미명 | O | 고정 18종 명칭 |
| 간단 설명 | O | 1~2문장 소개 |
| 카테고리 배지 | O | 운동형 / 지능형 / 예술형 |
| 북마크 버튼 | O (로그인 시) | 추가·해제 토글 |
| 상세 페이지 링크 | X | 별도 상세 URL 없음 |

#### (2) 카테고리별 페이지 `/category/[category]`

| 영역 | 사양 |
|------|------|
| **상단** | 선택 카테고리명 + 카테고리 소개 문구 |
| **본문** | 해당 카테고리 취미 **6개만** 그리드 나열 |
| **필터** | 동일 카테고리 내 탐색(필요 시 Top Bar와 연동) |

| URL 파라미터 | 값 |
|--------------|-----|
| `sports` | 운동형 |
| `intelligence` | 지능형 |
| `art` | 예술형 |

#### (3) 회원 관리

| 기능 | 사양 |
|------|------|
| **회원가입** `/signup` | 아이디(이메일)·비밀번호 입력, 서버 저장 시 **bcrypt** 해싱 |
| **로그인** `/login` | 아이디·비밀번호 인증, 세션 유지 |
| **로그아웃** | 세션 종료, 비회원 탐색 상태로 복귀 |
| **인증 방식** | NextAuth.js + Supabase `users` 테이블 |

#### (4) 마이페이지 `/mypage` (로그인 필수)

| 기능 | 사양 |
|------|------|
| **북마크 목록** | 사용자가 저장한 취미 카드 그리드 |
| **통계 차트** | 카테고리별 북마크 **개수·비율** (파이 또는 바 차트) |
| **비로그인 접근** | 로그인 페이지로 리다이렉트 또는 안내 |

### 3.2 기능별 접근 권한

| 기능 | 비회원 | 회원 |
|------|:------:|:----:|
| 홈·카테고리 탐색 | O | O |
| 카테고리 필터 | O | O |
| 북마크 추가/해제 | X | O |
| 마이페이지 | X | O |
| 북마크 통계 | X | O |

### 3.3 데이터·API (개발 사양)

| 리소스 | 저장소 | 비고 |
|--------|--------|------|
| 회원 | Supabase `users` | id, email, password(hash), created_at |
| 북마크 | Supabase `bookmarks` | user_id, hobby_id, created_at |
| 취미 마스터 | 프론트 고정 데이터 | 18종, DB 테이블 불필요 |

| API (예시) | 메서드 | 설명 |
|------------|--------|------|
| `/api/auth/signup` | POST | 회원가입 |
| `/api/auth/[...nextauth]` | GET/POST | 로그인·세션 |
| `/api/bookmarks` | GET | 북마크 목록 |
| `/api/bookmarks/toggle` | POST | 북마크 추가/해제 |
| `/api/bookmarks/stats` | GET | 카테고리별 통계 |

### 3.4 비기능 요구사항

| 항목 | 기준 |
|------|------|

| 보안 | 비밀번호 해싱, API 인증 세션 검증 |
| 배포 | Vercel 프로덕션 + 환경변수(Supabase, NextAuth) |

## 4. 사용자 페르소나 및 시나리오 (User Persona and Scenarios)

### 4.1 페르소나

| 항목 | Persona A (탐색형) | Persona B (저장형) |
|------|-------------------|-------------------|
| 이름 | 김민수 (28세, 직장인) | 이서연 (22세, 대학생) |
| 목표 | 주말에 할 만한 취미 찾기 | 관심 취미를 모아두고 비교 |
| 디바이스 | 모바일 + PC | 주로 PC |
| 행동 | 로그인 없이 먼저 둘러봄 | 북마크·통계 적극 사용 |

### 4.2 시나리오

#### 시나리오 1 — 비회원 탐색

1. 홈 접속 → Hero에서 서비스 목적 확인  
2. **예술형** 필터 선택 → 예술형 취미 6개만 그리드 표시  
3. Top Bar **예술형** 클릭 → 카테고리 페이지에서 소개 문구 + 6개 카드 확인  
4. 북마크 클릭 → 로그인 유도  

#### 시나리오 2 — 회원 가입·북마크

1. 회원가입(이메일·비밀번호) → 로그인  
2. 홈에서 **운동형** 필터 → 요가·수영 북마크  
3. **지능형** 카테고리 페이지 → 독서 북마크  
4. 마이페이지 → 북마크 3건 목록 + 운동형 2 / 지능형 1 통계 확인  

#### 시나리오 3 — 마이페이지 통계 활용

1. 로그인 상태로 마이페이지 진입  
2. 파이(또는 바) 차트로 카테고리별 비율 확인  
3. 부족한 카테고리 탭 이동 → 추가 북마크

## 5. 기술 스택 추천 (Technical Stack Recommendations)

| 영역 | 기술 | 선정 이유 |
|------|------|-----------|
| **Framework** | **Next.js 15** (App Router) | SSR/SSG, 라우팅, API Routes 통합 |
| **Language** | TypeScript | 타입 안정성, 유지보수 |
| **UI** | Tailwind CSS, shadcn/ui | 반응형·컴포넌트 일관성 |
| **Animation** | Framer Motion | Hero·전환 (선택 적용) |
| **Auth** | NextAuth.js | 세션·로그인/로그아웃 |
| **Database** | Supabase (PostgreSQL) | users, bookmarks 저장 |
| **Password** | bcryptjs | 비밀번호 단방향 해싱 |
| **Chart** | Recharts (또는 Chart.js) | 마이페이지 카테고리 통계 |
| **Deploy** | Vercel | Next.js 호스팅, CI/CD |
| **Package Manager** | npm | 프로젝트 표준 |

**환경변수 (필수 5종)**

| 변수 | 용도 |
|------|------|
| `NEXT_PUBLIC_SUPABASE_URL` | Supabase 프로젝트 URL |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | 클라이언트 anon key |
| `SUPABASE_SERVICE_ROLE_KEY` | 서버-side DB 접근 |
| `NEXTAUTH_SECRET` | 세션 암호화 |
| `NEXTAUTH_URL` | 배포 URL (로컬/프로덕션 구분) |

## 6. 제한사항 (Limitations)

| # | 제한 | 설명 |
|---|------|------|
| 1 | **취미 18종 고정** | 추가·삭제·편집 불가 (실습용) |
| 2 | **상세 페이지 없음** | 카드 클릭 시 별도 취미 상세 URL 없음 |
| 3 | **소셜 로그인 없음** | 이메일·비밀번호만 지원 |
| 4 | **북마크 비회원 불가** | 저장 기능은 로그인 후만 |
| 5 | **취미 검색/정렬** | 명세 범위 외 (구현 시 보너스) |
| 6 | **관리자·CMS 없음** | 콘텐츠는 코드 내 고정 데이터 |
| 7 | **다국어 미지원** | 한국어 UI만 |
| 8 | **실시간 협업·공유 없음** | 개인 북마크만 관리 |
